◆ devicePointFor()

Point View::RobotWorldCanvas::devicePointFor ( const Point &  aScreenPoint) const

The canvas is partially displayed in a scrolling window. Hence, the user viewable part of this window is probably just a part of this window. This function takes a point in the (visible) scrolling window coordinates and translates it to this canvas windows coordinates. It takes into account the number of lines and number of pixels between lines.

◆ handleNotification() [1/2]

void View::RobotWorldCanvas::handleNotification ( )
virtual

A Notifier will call this function if this Observer will handle the notifications of that Notifier. It is the responsibility of the Observer to filter any events it is interested in. This function should only be called from the main thread because wxWidgets does not allow for painting widgets in a background thread: all painting should be done in the main thread. Therefore all notifications are routed to handleBackGroundNotification() as a convenience.

Implements Base::Observer.
